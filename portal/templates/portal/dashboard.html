How this works: A public/private key pair is required to unlock mentorship questionnaire results. This requirement is mandatory and supports the project’s privacy-by-design commitment: while response records are technically stored by the platform, they are stored only in encrypted form, and private keys are not retained by the portal. As a result, platform operators are not in a position to directly access participants’ non-public answers.
Important before creating keys: If the private key is lost, encrypted mentorship responses cannot be recovered. Keep the private key in a safe place, with backup access restricted to trusted people.

If you are new to this process, prefer Generate new key pair. Use Insert existing public key only when you already have an external key-management process.
